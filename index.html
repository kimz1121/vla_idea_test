VLA + Skill Architecture

VLA Backbone과 Skill Detector Transformer를
분리해서 함께 사용하기

VLA가 매 프레임 Skill description을 처리하는 부담을 줄이고, Skill 미등록 상황에서도 일반화 능력을 보존하기 위한 통합 설계입니다. 단계별 흐름을 따라가며 어떻게 동작하는지 확인해 보세요.

⚡ VLA Backbone
FAST PATH · 15Hz · 큰 모델
이미지 + 언어만 입력으로 받습니다. Skill description은 처리하지 않으므로 매 프레임마다 가벼운 prefill로 끝나요. 항상 candidate로 등장하며, Skill이 적합하지 않은 상황에서 자연스럽게 winner가 됩니다.
Always-On Generalist
🔍 Skill Detector Transformer
SLOW PATH · ~3Hz · 작은 모델 · 가변 입력
공통 입력 + Skill description 토큰을 함께 받아 query-key 유사도를 계산합니다. Transformer 구조 덕분에 등록된 Skill 수에 관계없이 가변 길이 입력을 그대로 처리해요. 사전학습된 인코더(CLIP-style)로 zero-shot 동작이 목표입니다.
Open-Vocab Matcher
Unified Softmax — VLA를 candidate 중 하나로

기존 Skill 시스템은 항상 어떤 skill을 골라야 하는 closed-set 가정을 둡니다. 본 설계에서는 VLA backbone 자체를 candidate에 포함해 "지금은 어떤 skill도 적합하지 않다"는 결론을 자연스럽게 표현합니다. 슬라이더로 각 candidate의 logit을 조정하면서 softmax 분포가 어떻게 변하는지 보세요.

RAW LOGITS
SOFTMAX 분포
❌ Closed-set (기존)
항상 Skill 중 하나를 골라야 함. Skill 라이브러리에 적합한 게 없으면 잘못된 호출 강제 → generalization 손실
✅ Open-set (본 설계)
VLA가 candidate 중 하나. Skill이 적합하지 않으면 VLA가 winner → 사전학습 일반화 능력 보존
부가 효과 — 주파수 분리로 인한 추가 이점

Skill 호출 조건은 사람이 봐도 1초 단위로 판단 가능한 변화예요. Detector를 매 프레임 돌릴 필요가 없습니다. 1초 동안 VLA는 15번, Detector는 3번만 돌아도 충분하고, 그만큼 추가 비용이 줄어요.

통합 핵심 — 별도 detector + unified softmax 구조는 두 가지 결론을 동시에 만족합니다.

① 동적 확장 — Skill 등록은 detector의 description embedding 추가만으로 끝납니다. LLM이 새 tool을 등록할 때 LLM 자체를 fine-tune하지 않듯이.
② Generalization 보존 — VLA backbone을 explicit candidate로 명시해 "skill이 적합하지 않다"는 valid output을 자연스럽게 표현합니다. 의사결정 과정도 검증·디버깅 가능해져요.
Skill Hot-Swap

Skill을 추가/제거할 때
모델은 어떻게 반응할까?

Transformer의 가변 길이 입력 특성 덕분에 Skill을 핫 스왑할 수 있습니다 — 모델 가중치는 그대로 두고요. 직접 추가·제거하면서 토큰 시퀀스가 어떻게 변하는지 확인해 보세요.

Prefill 토큰 시퀀스 — 인터랙티브

모델이 매 추론에서 보는 Prefill 토큰 시퀀스입니다. 초록색은 고정 입력(이미지·언어), 노란색은 가변 Skill 토큰입니다. Skill 우상단의 ×로 제거하거나 +로 새 Skill을 추가해 보세요.

트레이드오프 — Skill 1개 ≈ 약 400 토큰. 200k 컨텍스트 창에서 Skill 100개도 20%정도 차지하지만, attention의 O(n²) 특성상 토큰이 늘면 추론 속도가 느려지고 Skill 간 혼동(tool hallucination)도 증가합니다.

그래서 "필요한 Skill만 골라서 등록"하는 전략이 중요합니다 — 페이지 1의 컨텍스트 오염 시뮬레이터가 정확히 이 트레이드오프를 보여줍니다. 재학습이 필요 없다는 건 큰 자유지만, 무한정 추가해도 좋다는 뜻은 아닙니다.
